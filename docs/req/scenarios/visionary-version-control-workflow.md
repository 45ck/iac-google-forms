# Scenario: Version Control Workflow (Visionary)

**Type:** Visionary (Future State with Tool)
**Persona:** Alex Chen (Platform Engineer)
**Goal:** Make form changes through a proper code review and CI/CD workflow

## Preconditions

- Form definitions stored in git repository
- Team uses GitHub for code review
- CI/CD pipeline set up with GitHub Actions
- Multiple environments: dev, staging, production

## Steps

| Step | Actor | Action | System Response |
|------|-------|--------|-----------------|
| 1 | Alex | Creates feature branch: `git checkout -b add-nps-question` | Branch created |
| 2 | Alex | Modifies `forms/customer-feedback.ts` | Adds NPS question |
| 3 | Alex | Runs `gforms diff` locally | Validates changes |
| 4 | Alex | Commits and pushes branch | Pushed to GitHub |
| 5 | CI | GitHub Action runs `gforms diff --ci` | Posts diff as PR comment |
| 6 | Teammate | Reviews PR, sees form changes | Approves PR |
| 7 | Alex | Merges PR to main | Merge complete |
| 8 | CI | GitHub Action deploys to production | Form updated |

## GitHub Actions Workflow

```yaml
# .github/workflows/forms.yml
name: Google Forms CI/CD

on:
  pull_request:
    paths:
      - 'forms/**'
  push:
    branches:
      - main
    paths:
      - 'forms/**'

jobs:
  diff:
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install gforms CLI
        run: npm install -g iac-google-forms

      - name: Diff forms
        env:
          GOOGLE_APPLICATION_CREDENTIALS: ${{ secrets.GOOGLE_CREDENTIALS }}
        run: |
          gforms diff forms/ --ci --output=diff.md

      - name: Comment on PR
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const diff = fs.readFileSync('diff.md', 'utf8');
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## Form Changes\n\n${diff}`
            });

  deploy:
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install gforms CLI
        run: npm install -g iac-google-forms

      - name: Deploy forms
        env:
          GOOGLE_APPLICATION_CREDENTIALS: ${{ secrets.GOOGLE_CREDENTIALS }}
        run: gforms deploy forms/ --yes
```

## Multi-Environment Deployment

```typescript
// forms/customer-feedback.ts
import { defineForm, env } from 'iac-google-forms';

export default defineForm({
  title: 'Customer Feedback',

  // Environment-specific configuration
  settings: {
    // Form ID per environment (created on first deploy)
    formId: env({
      dev: '1ABC-dev...',
      staging: '1ABC-staging...',
      prod: '1ABC-prod...',
    }),

    // Different notification recipients per env
    notifyEmails: env({
      dev: ['dev-team@company.com'],
      staging: ['qa-team@company.com'],
      prod: ['feedback@company.com', 'product@company.com'],
    }),
  },

  questions: [
    // ... same questions across all environments
  ],
});
```

```bash
# Deploy to specific environment
$ GFORMS_ENV=dev gforms deploy forms/
$ GFORMS_ENV=staging gforms deploy forms/
$ GFORMS_ENV=prod gforms deploy forms/
```

## Postconditions

- Form changes reviewed by teammate before production
- CI automatically shows what will change in PR
- Deployment only happens after merge to main
- Full audit trail in git history
- **Consistent across environments** - same TypeScript, different form IDs

## Benefits

| Aspect | Traditional | With IAC |
|--------|-------------|----------|
| Review | None | Full code review |
| Audit | None | Git history |
| Rollback | Manual recreation | `git revert` + deploy |
| Environments | Manual sync | Same code, different IDs |
| CI/CD | Not possible | Fully automated |
| Collaboration | Share URLs, hope for best | Standard PR workflow |
